Nivel 1: Capa Física

Resumen: La capa física, siendo la más básica del modelo OSI, se encarga de la transmisión de bits individuales de un nodo a otro.

Protocolos: Algunos protocolos que se utilizan en la capa física son Ethernet (IEEE 802.3), Wi-Fi (IEEE 802.11), Bluetooth (IEEE 802.15), entre otros.

Nivel 2: Capa de Enlace de Datos

Resumen: La capa de enlace de datos se ocupa de la entrega de datos de nodo a nodo. Su principal función es garantizar que la transferencia de datos sea libre de errores de un nodo a otro, sobre la capa física.

Protocolos: Algunos ejemplos de protocolos de la capa de enlace de datos incluyen Ethernet (IEEE 802.3), Token Ring, Frame Relay, entre otros.

Capa 3: Capa de Red

Resumen: La capa de red proporciona los medios funcionales y de procedimiento para transferir secuencias de datos de longitud variable de un host fuente a un host de destino a través de una o más redes, manteniendo las funciones de calidad de servicio.

Protocolos: Algunos ejemplos de protocolos de la capa de red incluyen IP (Internet Protocol), ICMP (Internet Control Message Protocol), IGMP (Internet Group Management Protocol), IPsec (Internet Protocol Security), entre otros.

Capa 4: Capa de Transporte

Resumen: La capa de transporte asegura la entrega completa y confiable de paquetes de datos. Proporciona mecanismos como el control de errores, el control de flujo y el control de congestión para realizar un seguimiento de los paquetes de datos, verificar errores y duplicaciones, y reenviar la información que falla en la entrega.

Protocolos: Algunos ejemplos de protocolos de la capa de transporte incluyen TCP (Transmission Control Protocol), UDP (User Datagram Protocol), DCCP (Datagram Congestion Control Protocol), entre otros.

Capa 5: Capa de Sesión

Resumen: La capa de sesión controla las conversaciones entre diferentes computadoras. Una sesión o conexión entre máquinas se establece, se gestiona y se termina en la capa 5.

Protocolos: Algunos ejemplos de protocolos de la capa de sesión incluyen NFS, SQL, RPC, etc.

Capa 6: Capa de Presentación

Resumen: La capa de presentación se encarga de la representación de la información. Asegura que la información que envía la capa de aplicación de un sistema puede ser leída por la capa de aplicación de otro sistema.

Protocolos: Algunos ejemplos de protocolos de la capa de presentación incluyen TLS, SSL, etc.

Capa 7: Capa de Aplicación

Resumen: La capa de aplicación es la capa del modelo OSI más cercana al usuario final. Esta capa interactúa directamente con el software de aplicación para realizar las funciones de red.

Protocolos: Algunos ejemplos de protocolos de la capa de aplicación incluyen HTTP, FTP, SMTP, DNS, etc.
